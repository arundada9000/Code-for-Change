🔐 Authentication Flows

Password Login
1. POST /api/auth/login { email, password }
2. Find user → check isActive, isVerified, lockUntil
3. Compare bcrypt password
4. On fail: increment failedLoginAttempts → lock at 5
5. On success: reset attempts, update lastLogin, push loginHistory
6. Generate JWT, set HttpOnly cookie + return token in body
7. Frontend stores token in localStorage (fallback), user in localStorage
Rate: 5/15minJWT: 1dCookie: HttpOnly
WebAuthn / Passkey Login
1. POST /webauthn/login-options → get challenge
2. Browser: startAuthentication() → biometric prompt
3. POST /webauthn/login-verify { challengeId, response }
4. Find user by credentialId → verify signature with stored public key
5. Same security checks: isActive, isVerified, lockUntil
6. Update counter (replay protection), update lastLogin
7. Issue SAME JWT + cookie as password login
8. Frontend stores token + user (identical to password login)
Rate: 10/15minChallenge: 60s TTLIn-memory store
